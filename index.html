Donate by Necessity

Ruby on Rails

Donate By Necessity

Application Description

  • A real time tracking donation platform that helps maximize every dollar.
  • Contains a donor interface where a user can manage their their individual donations.
  • The ability to add a recipient to the system, or donate to an exisitng recipient added by another user.
  • Donations are maximized by allowing the donor to be very specific with where they want to donate their money.
  • Real time tracking allows for a plethora of metrics to enhance the user experience.

Project Requirements

  • Model had to include a has_many, belongs_to and a has_many :through relationship.
  • Model had to include reasonable validations.
  • Included class level ActiveRecord scope methods.
  • Allowed for standard user authentication, including OmniAuth functionality.
  • All forms were required to correctly display validation errors.
  • And of course, it had to be DRY.

Freight Tracker

Sintra Framework - MVC Pattern

Freight Tracker

Application Description

  • A simple freight tracking app that shows good use of RESTful API.
  • Contains a UI that allows for simple tracking of specific freight, with the ability to add an associated shipping route.
  • It allows the user to stategically place in freight information when necessary, and then gives them the ability to retroactively add in a route at any given time.

Project Requirements

  • Model had to include ActiveRecord for database mapping.
  • A has_many relationship, and a belongs_to relationship were required on at least two models.
  • User login capability with exclusive user priveleges was necessary for completion.
  • Display of CRUD functionality with the database.

TVGuide.com Web Scraper

Object Oriented Ruby - Ruby Gem

Web Scraper

Application Description

  • A simple web scraper that gives the user a nifty way to check what sports are playing that day.
  • It's an efficient, lightweight Ruby gem that any sports fan would enjoy.
  • The CLI interface gives 10 sports categories to choose from, and is constantly updated everyday via the data from www.tvguide.com.

Project Requirements

  • Had to include a CLI.
  • Model had to include access to data from a web page.
  • Data had to go at least one level deep. "Level", being a user can make a choice and then get more information from that choice.
  • Had to use good OO design patterns.
  • Instead of using a collection of hashes to store a data, we were required to use a collection of objects.
